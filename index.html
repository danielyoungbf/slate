    Device Component API

    Compile Test Fixture

    To compile a test fixture: /compile-test-fixture

    {
    "TestFixtureId": 123,
    "FormattedName": "ArchiveFixture",
    "PythonScript": "from Helpers.TestCase import TestCase\n..."
}

    Test fixtures are always uniquely identified within the framework by their FormattedName, hence this value should always be unique.

    The PythonScript must contain syntactically valid python, otherwise this call will fail.

    While the script is compiling it will make regular regular request to the CompilationResult endpoint on the server. This will contain screen data and error messages if any.

    Return Status

    • BadRequest if the parameters couldn't be validated.
    • UnprocessableEntity if a fixture with the same name already exists.
    • OK Otherwise

    Run Test Fixture

    To run a test fixture: /run-test-fixture

    {
    "TestFixtureId": 123,
    "FormattedName": "ArchiveFixture",
    "RunId": 42
}

    The fixture must exists on the device before this endpoint can be called.

    Once an error has been encountered or the fixture completes a request will be made to the FixtureReport endpoint on the server.

    Return Status

    • BadRequest if the test fixture doesn't exist
    • OK Otherwise

    Delete Test Fixture

    To delete a test fixture: /delete-test-fixture

    {
    "TestFixtureId": 123,
    "FormattedName": "ArchiveFixture"
}

    Return Status

    • BadRequest if the test fixture doesn't exist
    • OK Otherwise

    Deploy Test Fixture

    To deploy a test fixture: /deploy-test-fixture

    {
    "TestFixtureId": 123,
    "FormattedName": "ArchiveFixture",
    "PythonScript": "from Helpers.TestCase import TestCase\n...",
    "Hashes": {
        "Tests/ArchiveFixture/Screenshots/Test/PAGE_0": "ksadjfklajslkfjaklsjdfjaij23fjwjfwkj",
        "Tests/ArchiveFixture/Screenshots/Test/PAGE_1": "ksadjfklajslkfjaklsjdfjaij23fjwjfwkj"
    }
}

    Deploys an already compiled fixture to the device.

    Return Status

    • BadRequest if the test fixture exists
    • OK Otherwise

    List Test Fixtures

    To list test fixtures: /list-test-fixtures

    {
    "TestFixtures": [
        "ArchiveFixture",
        "GuiFixture"
    ]
}

    Returns a list of test fixtures currently on the device.
